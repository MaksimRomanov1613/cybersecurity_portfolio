На основе описания сети из файла not_really_typical_org вручную составил таблицу инвентаризации My_inventory.xlsx. 
Для каждого устройства указал установленную ОС с версией, как минимум одно приложение, соответствующий функционалу, IP-адрес и маску подсети, доменные и локальные учётные записи, а также принадлежность к домену. 
Это научило вычленять из разрозненной информации полный перечень активов, фиксировать критически важные параметры и готовить данные для последующего аудита.

Чтобы приблизить задачу к реальной работе с большими инфраструктурами, дополнительно отработал взаимодействие с AI-инструментами. 
С помощью AI сгенерировал шаблон таблицы, сравнил его с требованиями и доработал. 
Затем попросил AI проанализировать описание сети и найти скрытые активы, которые можно упустить - например, сетевые принтеры, ИБП, дополнительные коммутаторы. 
Добавил их в инвентаризацию. Наконец, написал Python-скрипт check_inventory.py, проверяющий наличие для каждого устройства IPv4-адреса, версии ОС и установленного ПО для не-сетевых узлов. 
Это позволило автоматизировать контроль полноты данных и натренировать навык использования AI для рутинных операций без потери экспертной оценки.
