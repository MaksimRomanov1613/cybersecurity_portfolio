Инструкция по запуску и тестированию

1. Установите nmap (обычно уже есть в Kali):

bash
sudo apt update && sudo apt install nmap -y

2. Запустите уязвимый тестовый сервер MongoDB (например, версия 4.0 без пароля):

bash
docker rm -f mongo-vuln 2>/dev/null
docker run -d --name mongo-vuln -p 27017:27017 --tmpfs /data/db mongo:4.0

3. Проверьте, что контейнер работает:

bash
docker ps
В столбце STATUS должно быть Up.

4. Запустите сканер для локальной проверки:

bash
python scan.py 127.0.0.1/32
Вы должны увидеть список баз данных (обычно admin, config, local).

5. Для сканирования реальной сети передайте CIDR-адрес (виртуальная машина должна быть в режиме «Сетевой мост»):

bash
python scan.py 192.168.1.0/24
Примечание
Если docker не установлен, вы можете использовать любой другой сервер MongoDB без аутентификации на порту 27017.

Скрипт выводит только хосты, где обнаружены базы данных без пароля.

Инструкция по запуску и тестированию (локально)

1. Убедитесь, что установлен nmap (в Kali он есть по умолчанию):

bash
sudo apt update && sudo apt install nmap -y

2. Запустите уязвимый тестовый сервер MongoDB (версия 4.0 без пароля) через Docker:

bash
docker rm -f mongo-vuln 2>/dev/null
docker run -d --name mongo-vuln -p 27017:27017 --tmpfs /data/db mongo:4.0

3. Проверьте, что контейнер работает:

bash
docker ps
В столбце STATUS должно быть Up.

4. Запустите сканер локально:

bash
python scan.py 127.0.0.1/32
Вы увидите список баз данных (admin, config, local), если всё настроено верно.
